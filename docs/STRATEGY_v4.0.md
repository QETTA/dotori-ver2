## 유보통합 대응 (v4.0 핵심 추가)

### 배경
2026년 유보통합 정책으로 어린이집과 유치원이 하나의 관리 체계로 통합됨.
도토리는 **통합 이전부터 양쪽을 아우르는 유일한 민간 플랫폼**으로 선제 대응.

### 용어 통일
- 공식 용어: **보육·교육기관** (어린이집·유치원 통칭)
- 앱 내 표기: "어린이집·유치원" 병기, 맥락에 따라 "시설" 약칭
- 데이터 모델: `Facility.type` — `어린이집 | 유치원` (이미 지원)

### 경쟁우위: 유보통합 선제 대응
| 차별점 | 설명 |
|--------|------|
| **통합 데이터** | 어린이집(정보공개포털) + 유치원(유치원알리미) 단일 검색 |
| **통합 TO 예측** | 어린이집·유치원 공통 TO 계산 공식 |
| **통합 전자서명** | 어린이집 입소동의서 + 유치원 입학동의서 동일 플로우 |
| **AI 인텐트** | "유치원 찾아줘", "유보통합 뭐야" 등 유치원 키워드 완전 커버 |

## 5개 공공 API

| # | API | 대상 | 데이터 | 인증 |
|---|-----|------|--------|------|
| 1 | **어린이집정보공개포털** (info.childcare.go.kr) | 27,387개소 | REST+SHEET, 정원/현원/교사 | 개발계정 심의 |
| 2 | **유치원알리미** (e-childschoolinfo.moe.go.kr) | 8,500개소 | REST, 원비/교원/방과후 | SNS 로그인 즉시 |
| 3 | **전국어린이집표준데이터** (data.go.kr) | 27,387개소 | 정원/현원/위경도/CCTV | Open API 자동승인 |
| 4 | **행정안전부 주민등록 인구통계** | 전국 행정동 | 0~5세 연령별 인구 (매월) | 공공데이터포털 |
| 5 | **KOSIS OpenAPI** (통계청) | 전국 | 출생아수, 출산율, 맞벌이가구 | API 키 |

## 3개 공개 데이터셋

- **서울 열린데이터광장** — 연령별 반수+아동수 (0세반 정원9/현원8→TO1) = MVP 핵심
- **인천 남동구 입소대기현황** — 대기자수 선례 → 공공데이터법 27조로 타 지자체 신청 가능
- **서울시보육포털** (iseoul) — 실시간 정원/대기자, 긴급보육, 시간제

## TO 예측 엔진

```
TO_예측 = 정원 - 현원 + 예상졸업 - 예상수요(행정동 인구) × 시설매력도

Layer 1: 시설정보 (API 즉시)
Layer 2: TO 계산 (서울 즉시, 전국 신청 후)
Layer 3: 예측 변수 (KOSIS + 행안부)
Layer 4: 독자 데이터 (원장 입력, 리뷰, 토리챗 로그)
```

**특허 가능성**: 수용가능성 상태 벡터 (시설 매력도 + 지역 수요 + 계절 보정)

## 전자서명 시스템

- **대상 문서 7종**: 입소동의서, 개인정보동의서, 귀가동의서, 투약의뢰서, 현장학습동의서, 차량운행동의서, CCTV열람동의서 + 근로계약서
- **하이브리드 방식**: 일반 동의서 = 자체(Canvas + PDF), 법적 문서 = 모두싸인 API
- **감사추적인증서**(Audit Trail) = 전자문서법 제4조 법적 효력
- **퍼널 단축**: 탐색→상세→관심→견학→입소계약→서류 일괄서명→완료 (기존 2~3시간 → **10분**)

## 정부 정책 레버리지 (2026)

- 고용노동부 HR 무료 지원 (30인 미만 = 어린이집 대부분)
- 보육사업안내 개정 (야간연장/24시간/반편성)
- **유보통합 가속 → 어린이집+유치원 통합 관리 = 도토리의 핵심 기회**
- 어린이집 평가제 + 유치원 평가 → 전자서명 감사추적 = 완벽한 증빙
